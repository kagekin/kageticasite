# 原稿進行コンシェルジュ 実装計画（アップデート）

## 概要

同人誌や個人制作の原稿の進捗を管理し、キャラクター（コンシェルジュ）がサポートしてくれるWebアプリケーションの開発計画です。

当初は React/Firebase 構成を検討していましたが、**GitHub Pagesでの手軽な公開やオフラインでの動作を優先し、単一の HTML ファイル (`genko55.html`) 内に HTML, Tailwind CSS, Vanilla JavaScript を記述し、データ保存に `localStorage` を用いる構成**に変更して実装を完了しています。

## 実装済みの内容とアプローチ

### 1. アプリケーションアーキテクチャの変更

- **脱Firebase化**: データ保存を Firestore からブラウザの `localStorage` に変更。サーバーサイド処理をなくし、GitHub Pages だけで完全に機能する静的サイト構成にしました。
- 単一HTMLでのSPA的挙動: JavaScriptでDOMを描画し、ダッシュボード・プロジェクト設定・コンシェルジュ設定の3画面をシームレスに切り替えます。

### 2. 進捗評価とステータス判定の高度化

- **目標ペースの導入**: プロジェクト設定でユーザーが「目標ペース（1日あたり/週あたり等）」を設定できるようにし、それに応じて「順調 (Good)」「ピンチ (Panic)」の判定を行うようにしました。
- **直感的な進捗表示**: 各工程（下書き、ペン入れなど）の進捗を平均化して「全体の残り 〇〇％」として表示するように変更し、架空のページ数換算による混乱を解消しました。

### 3. コンシェルジュ（キャラクター）機能の拡張

- **吉日・占いメッセージ機能**: コンシェルジュ設定から「吉日メッセージ」をカスタム登録できるようにしました。特定の確率（シード値に基づく）で、タップ時にデフォルト画像を維持したままこのメッセージが表示されます。
- **クリエイター向け豆知識の追加**: 座りすぎ防止や集中力アップのコツなど、クリエイターの健康に配慮したデフォルトメッセージを追加しました。
- **画像リセット機能**: アップロードしたカスタムのキャラ画像を、各ステートごとにデフォルト状態に戻せるボタンを追加しました。

### 4. 操作性・UIの改善

- **チェックリストのトグル化**: 進捗のプラス/マイナスボタンだけでなく、チェックをつけることで一気に100% (MAX) にできる機能を追加しました。
- **しめきり直前の強調表示**: 残り日数が7日以下になった場合、残り日数の表示が赤文字で点滅し、アラートアイコンが表示されるカウントダウン演出を追加しました。
- **設定変更の即時反映**: 設定画面でしめきりを延ばすなどの変更を行った後、「保存して戻る」を押した瞬間に、コンシェルジュのセリフと表情が最新の状況に合わせて正しく再計算・リセットされるようバグ修正を行いました。

### 5. 外部連携とコミュニティ要素

- **プリセットキャラクター募集枠**: 画面下部に、初期設定キャラクターを描いてくれるボランティアを募集する告知エリアを新設しました。応募先のX(Twitter)へのリンク (`https://x.com/kagetica`) を配置しました。
- **X(Twitter)進捗シェア機能**: 現在の残り日数、各工程の状況、現在のペースなどをまとめてツイートできるシェアボタンを実装しました。
