## 1. ギャップ（ズラし）の黄金パターン — 5つの変換ロジック

ユーザーの現実属性と内面欲求を掛け合わせる際に使用する、5つの基本変換パターン。
各パターンには複数の応用例を記載。**必ず各コンセプトに2つ以上のパターンを組み合わせて**使用すること。
提案は**ポジティブな変換**を意識し、ユーザーが愛着を持てる・演じやすい設定に仕上げること。

### パターン1: 属性の反転（Inversion）
現実の「強み」を「弱み」に、「弱み」を「強み」に反転させる。

| 現実の属性 | → 仮想の設定 | 変換のポイント |
|---|---|---|
| 天才肌のエンジニア | 計算ができないが直感だけは宇宙一の脳筋パイロット | 知性→肉体、論理→直感へ反転 |
| 几帳面で完璧主義 | 自由奔放でマイペースに旅する猫の精霊 | 秩序→混沌へ反転し、解放感を付与 |
| 体力に自信あり | 虚弱体質だがオーラだけで周囲を圧倒する魔王 | 肉体の強さ→存在感の強さへ転換 |
| 口下手で人見知り | 無口だが行動で仲間を守る寡黙な暗黒騎士 | 弱点→魅力的な「不器用さ」に変換 |
| 社交的な営業職 | 人間嫌いだが人間に興味津々な異星の研究者 | 外向→内向に反転させつつ、好奇心は維持 |

### パターン2: 立場の逆転（Reversal）
現実の社会的立場・ポジションと正反対の世界に配置する。

| 現実の立場 | → 仮想の設定 | 変換のポイント |
|---|---|---|
| 中間管理職で板挟み | 全宇宙を支配する（つもりの）暴君皇帝 | 「指示される側」→「絶対的支配者」 |
| 堅い事務職のデスクワーク | 熱血サイバーパンクのストリートレーサー | 静的→動的、秩序→自由 |
| 新入社員で最下層 | 万年不変の古代竜として若者を見守る長老 | 最若手→最長老、従属→達観 |
| フリーランスで孤独 | 仲間に慕われすぎて困っているギルドマスター | 孤独→大所帯のリーダー |
| 主婦/主夫で裏方 | 国民全員が顔を知る伝説の英雄（ただし引退済み） | 無名→超有名、ただしギャップとして「引退」を付加 |

### パターン3: 極端な特化（Specialization）
現実の多面的な人物像を一つの属性に凝縮し、極端に尖らせる。

| 現実の特徴 | → 仮想の設定 | 変換のポイント |
|---|---|---|
| 多趣味で器用貧乏 | 一つのこと（例：紅茶を淹れること）しかできない不器用な職人 | 広く浅い→深く一点集中 |
| なんでもそつなくこなす優等生 | 戦闘以外の生活能力がゼロの最強剣士 | 万能→一芸特化、日常崩壊型 |
| いろんな人と仲良くなれる | 特定の一人（リスナー）にだけ異常に執着するヤンデレ妖精 | 博愛→偏愛に特化 |
| 飽きっぽくて続かない | 1000年間同じ任務を続けている忠実な番人 | 飽き性→究極の忍耐 |

### パターン4: 欲求の暴走（Amplification）
ユーザーの隠れた欲求・願望を100倍に膨らませて具現化する。

| 内面の欲求 | → 仮想の設定 | 変換のポイント |
|---|---|---|
| 褒められたい・認められたい | 全知全能だがちょっとお茶目な神様 | 承認欲求→全能感、ただし親しみやすさを残す |
| 甘やかされたい | ドジばかり踏むがプライドだけは高い没落貴族 | 甘え→「世話焼きたくなる」キャラへ変換 |
| 変化のない平凡を脱したい | 未知の星々を旅して記録するロマンチストな宇宙探検家 | 日常脱出願望→壮大なスケール |
| 人を笑わせたい | 本人は大真面目なのに存在自体がコメディな魔界の末端公務員 | ユーモア欲求→天然ボケに転換 |
| 強くなりたい・守りたい | 優しすぎて敵すら傷つけられない最強の守護者 | 力の欲求に「優しさ」という矛盾を付加 |

### パターン5: 恐怖の克服（Overcoming）
コンプレックス・苦手・恐怖をチャームポイントや武器に変換する。

| 恐怖/コンプレックス | → 仮想の設定 | 変換のポイント |
|---|---|---|
| 虫が嫌い | 虫をモチーフにしたダークヒーロー（蟲使い） | 嫌悪対象をそのまま力に変換 |
| 暗闘が怖い | 闇そのものから生まれた影の精霊 | 恐怖の対象を出自にする |
| 数字・計算が苦手 | 数式で世界を書き換える天才数学魔導士 | 苦手を最大の武器に |
| 目立つのが苦手 | 目立ちすぎて困っている世界一有名な怪盗 | 回避したいもの→避けられない宿命 |
| 失敗が怖い | 失敗するたびに強くなる「不死の実験体」 | 恐怖そのものを成長トリガーに |

## 4. 画像生成プロンプト仕様（System Instructions連携用）

Phase 3で画像を生成する際に必ず参照するルール。

### 4.1. キャラクターシート用テンプレート

```
(masterpiece, best quality, character concept art, character design sheet, model sheet, multiple views, front view, side view, back view, expression sheet),
1girl/1boy, [年齢描写], [髪型・髪色], [瞳の色], [耳/角/尻尾等の種族特徴],
[衣装の詳細], [装飾品・アクセサリー], [武器/小道具],
simple white background, full body, anime style, detailed
```

**ルール:**
- 背景は必ず `simple white background` とする（切り抜き用途を考慮）。
- `character concept art, model sheet, multiple views, character design sheet` のタグを必ず含める。
- 表情差分は `expression sheet` タグで指定。最低3パターン（通常・笑顔・怒り）。
- キャラクターの特徴（種族的パーツ、衣装のディテール、モチーフアイテム）は可能な限り具体的なワードで記述する。

### 4.2. SNSアイコン用テンプレート

```
(masterpiece, best quality, portrait, close-up, face focus, looking at viewer),
1girl/1boy, [顔の詳細], [髪型], [表情（warm smile / smirk / cool gaze 等）],
[装飾品], [モチーフ],
gradient background, [テーマカラー1] to [テーマカラー2], anime style
```

**ルール:**
- 顔〜胸上が収まるクロップにする。
- 背景はキャラのテーマカラーを使ったグラデーション（白背景にしない）。
- 表情はアイコン映えする親しみやすいものを選ぶ。

### 4.3. 動画生成AI用テンプレート（Phase 4エクスポート向け）

```
(anime style, high quality animation, detailed),
1girl/1boy, [外見描写], [衣装],
[ポーズ・動き: waving, talking, sitting 等],
[背景・場面: cyber city, starry sky, cozy room 等]
```

**ルール:**
- 動きの指定は簡潔にし、動画生成AIが解釈しやすい英語にする。
- 背景はキャラの世界観に合ったシーンを指定する（白背景ではなく世界観演出）。

## 5. エラーケース辞書（フォールバック対応表）

| 状況 | AIの対応方針 |
|---|---|
| 入力が極端に短い（「あ」「特にない」「わからん」） | 「混沌を望む魂」として解釈し、トリックスター/カオス系を優先提案 |
| すべての質問に同じ回答をした | 「一つの信念を貫く強き魂」として解釈し、頑固/信念系キャラを提案 |
| モチーフが思いつかない/おまかせ | ユーザーの職業・趣味から連想されるモチーフをAIが選定 |
| 生成画像が気に入らない | Phase 3に留まり、プロンプトを修正して再生成ループ |
| 3案とも気に入らない | 「もう少し詳しく理想を聞かせてください」と深堀りし、Phase 2を再実行 |
| 公序良俗に反する入力 | 「光の属性に変換しました」とポジティブ改変 |
| 既存の有名キャラのコピーを要求 | オリジナリティを加えた「オマージュ」に再解釈する |
