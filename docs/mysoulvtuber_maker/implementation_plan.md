# 実装計画: MySoulVTuber Maker MVP

## 1. 概要
「MySoulVTuber Maker」初期リリース（MVP）向けの、フロントエンド（HTML/CSS/Vanilla JS）実装計画です。ユーザーの深層心理や欲求を入力させ、対話AI（Gemini等）向けの「最適化されたプロンプト」を自動生成するUI/UXを構築します。
提供された「ギャップの黄金パターン」や「AIステートマシン設計」を基に、ユーザーが迷わずに自己設計〜プロンプトコピー〜外部ツール（OiiOiiAI等）への持ち出しを行える導線を提供します。

## 2. ユーザー確認事項 (User Review Required)
> [!IMPORTANT]  
> MVPではバックエンド（APIキーの保護など）を持たない静的HTMLとして実装予定です。「Gemini APIとの通信」ではなく、**「ユーザーの入力内容を組み立てて、Gemini / 専用Gemに貼り付けるための高度なプロンプトを生成し、ワンクリックでコピーさせる方式（または専用Gemへの誘導と入力データの受け渡し）」**を採用してよろしいでしょうか？（これにより、キー漏洩リスクなし・サーバー維持費ゼロでMVP運用が可能となります）。

## 3. 提案する変更内容

### 3.1. フロントエンド (`lab/` ディレクトリ配下)

#### [NEW] `lab/mysoulvtuber.html` (または `index.html`)
ユーザーがアクセスするメインのWebアプリケーションUI。「オルタナティブエゴ」を想起させる、少しミステリアスかつサイバーなデザインの画面を構築。
- **Step 1 (Phase 0):** 現実のプロフィール（職業、趣味）と、VTuberに残したい要素の入力。
- **Step 2 (Phase 1):** 魂の解析テスト。MBTI的な情報処理傾向、承認欲求の方向性、倫理観に関する3つの質問。
- **Step 3 (Prompt Generation):** 入力項目から「専用AI（Gemini等）に入力するためのメタプロンプト」を生成し、コピーボタンを表示。
- **Step 4 (Phase 4 相当):** AI（Gemini等）で生成されたテキストや画像を元に、動画生成（OiiOiiAI）やモデラー等次のステップへ進むためのガイド表示。

#### [NEW] `lab/mysoulvtuber.css`
- ダークテーマを基調とし、サイバーパンク/心理分析ツールらしさを感じさせるスタイリング。
- バニラCSSを使用し、レスポンシブ対応。スムーズな画面遷移アニメーション（Fade-in, slide-up等）を付与してUXを向上させる。

#### [NEW] `lab/mysoulvtuber.js`
- 入力フォームの状態管理とバリデーション。
- プロンプト生成ロジックの実装（ユーザーの回答内容を埋め込んだ、長文のシステム指示書の構築）。
- クリップボードへのコピー機能の実装。
- ステップ間のUI切り替え制御。

## 4. 検証計画 (Verification Plan)
### 手動検証 (Manual Verification)
1. `lab/mysoulvtuber.html` をローカルブラウザで開き、デザインとレイアウト崩れがないか確認。
2. フォームに一通り入力し、「次へ」のアニメーションや動作が適切か確認。
3. 生成されるプロンプトが要件定義書（ギャップ生成、アセット出力指示）に沿った内容になっているか確認。
4. プロンプトのコピーボタンが正しく作動するか確認。
5. （可能であれば）コピーしたプロンプトを実際にGemini等に入力し、期待通りの出力（3つのコンセプト提案や画像生成プロンプトの出力）が得られるかをテスト。
